The Filch CLI

A User Journey: From Setup to Leaderboard

This infographic visualizes a teacher's 5-day workflow using a powerful command-line tool designed for efficient attendance tracking.

Phase 1: Day 0 - The Setup

Before tracking attendance, Professor Tausif performs a quick, one-time setup to create his classes and add his students in bulk, laying the foundation for the week.

filch create class "10-A"
↓
filch create class "10-B"
↓
filch insert into "10-A" --rolls "[1,2,3]" --names '["Rohan", "Anya", "Zoya"]'

Setup Workflow

This simple, sequential process prepares the entire system in minutes. The key steps are:

  • Create Classes: Professor Tausif establishes the containers for his student rosters, "10-A" and "10-B".
  • Bulk Insert Students: Using a single, powerful command, he populates an entire class roster by providing lists of roll numbers and names. This saves significant time over manual, one-by-one entry.

Phase 2: The Daily Workflow

For the next 5 days, the professor follows a simple, repeatable 3-step process to track attendance for each of his classes.

Step 1: Start a New Day

Each morning begins with the `start newday` command, which prepares the system to log attendance for the current date.

filch start newday

Step 2: Select the Class

Before taking attendance, he selects the active class. The tool remembers this choice for subsequent commands.

filch select class "10-A"

Step 3: Mark Attendance

He marks each student by their unique roll number as Present (P) or Absent (A).

filch mark 1 P
filch mark 2 A

Phase 3: The Report

After 5 days of tracking, Professor Tausif runs the `leaderboard` command on the selected class to instantly see the final results and student rankings.
